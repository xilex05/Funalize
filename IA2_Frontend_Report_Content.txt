3. PROJECT ABSTRACT
============================================================
Funalize is a full-stack web application developed to help groups make quick and fair party-related decisions. The platform allows users to create or join a party room using a unique party code and collaborate in real time while selecting options in different categories (Food, Games, and Music).

The application supports two primary decision modes:
1. Voting mode, where members vote for their preferred options (up to two votes per user in the active category).
2. Random mode, where a random picker animation selects a final option from the active category.

The system also includes role-based control, where the host manages category and mode transitions, while all participants can view updates instantly through Socket.IO events. Additional features include secure login/signup using JWT authentication, live option updates, real-time member list synchronization, and an in-room chat system.

Technologies used:
- Frontend: React (Vite), React Router, Bootstrap Icons, custom CSS
- Backend: Node.js, Express.js, Socket.IO
- Database: MongoDB with Mongoose
- Authentication: JWT + bcrypt

The project demonstrates practical implementation of real-time full-stack architecture, collaborative UI state management, and secure REST API design.

4.2 Main Functional Modules
1. Authentication Module
- User registration and login
- Password hashing with bcrypt
- JWT token generation and protected routes

2. Party Room Module
- Create party and generate unique party code
- Join party by code
- Real-time member synchronization

3. Option Management Module
- Add/delete options in active category
- Enforce limit of 3 options per user
- Lock editing when a mode is active (non-host users)

4. Decision Engine Module
- Voting mode with max 2 votes per user
- Finalize voting to compute winner(s)
- Random mode with animated picker and winner reveal

5. Real-Time Communication Module
- Live updates via Socket.IO (category changes, options updates, mode updates, finalized results, chat updates)

6. Chat Module
- In-party chat for members
- Message history persisted in database with sender details and timestamp

4.3 Data Flow (Simplified)
1. User logs in and gets JWT.
2. User creates/joins party room.
3. Frontend sends REST requests for actions (add option, vote, mode change).
4. Backend updates MongoDB and emits Socket.IO events to party room.
5. All connected clients instantly update UI state.

8. CONCLUSION
============================================================
Funalize successfully demonstrates the practical use of MERN-stack technologies to build an interactive, real-time collaborative web application. The system solves a common group problem (deciding activities/items) by combining structured workflows (voting) with lightweight spontaneity (random mode).

The project highlights key full-stack competencies such as secure authentication, REST API design, real-time bidirectional communication, role-based controls, and dynamic UI state management. The final outcome is a usable and visually engaging platform that supports multi-user collaboration with low latency.

Future improvements can include analytics dashboard, stronger moderation controls, media sharing in chat, deployment to cloud infrastructure, and automated testing coverage.
